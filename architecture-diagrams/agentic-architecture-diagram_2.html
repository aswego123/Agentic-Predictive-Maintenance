New agentic mechanisms added in this revision

Orchestrator control/memory bus (indigo) — previously the Orchestrator only fired once at the top of the cycle and had no visible link to the individual agents. It now stays connected throughout: it injects fleet-memory context into Physics, ML, Judge, Calibration, and Action agents, and each of them can report status back. Solid gray arrows are the primary data handoff between agents; this indigo bus is separate — it's coordination and context, not the analysis result itself.
Calibration loop, gated by Engineer Approval (default for all assets) — the Calibration Agent proposes a parameter change, but every re-run of the Digital Twin / Simulation Layer is reviewed and approved by the engineer first, up to 5 rounds, until Judge confidence ≥ 95%. If it never converges, the cycle stops and is flagged for manual investigation instead of looping indefinitely. This keeps a human aware of every simulation-input change, not just the final outcome.
Judge self-critique — before finalizing a verdict, the Judge Agent reviews its own divergence classification against the asset's criticality and historical false-alarm rate, and revises if unjustified.
Physics ↔ ML negotiation — Physics Agent and ML Correction Agent exchange up to 2 rounds of reconsideration directly with each other before handing off to the Judge, narrowing disagreement early.
